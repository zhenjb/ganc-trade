P5 Task — Wallet-Signed MsgDeposit

Implement real deposit from the Next.js frontend. Backend does not sign deposit transactions.

Goal

Frontend wallet → sign MsgDeposit → broadcast to chain → refresh backend state

Use this message

typeUrl: /ob.zkdex.v1.MsgDeposit

value: {
  creator: connectedWalletAddress,
  denom: "uusdc",
  amount: "100"
}

Required setup

  • Connect Keplr/Leap wallet.
  • Add/suggest local chain if needed.
  • Use chain config from P1: chainId, rpc, rest, bech32Prefix, feeDenom, gas/fee.
  • Register/use custom proto message /ob.zkdex.v1.MsgDeposit.

Implementation flow

  1. User connects wallet.
  2. FE gets wallet address.
  3. User inputs denom and amount.
  4. FE builds MsgDeposit.
  5. Wallet signs and broadcasts tx.
  6. FE receives txHash.
  7. FE parses deposit_id from EventDeposit if available.
  8. FE refreshes backend state.

After broadcast

If deposit_id is available:

GET /api/deposits/{depositId}
GET /api/state

If not available yet, show txHash and refresh:

GET /api/state

Important notes

  • Do not call POST /api/deposit in real wallet mode. It is local/dev-only.
  • Use /ob.zkdex.v1.MsgDeposit, not ob.backend MsgDeposit.
  • EventDeposit fields to parse: deposit_id, creator, denom, amount.

Done when

  • User can sign and broadcast a real MsgDeposit.
  • Chain creates a deposit record.
  • FE can refresh GET /api/state.
  • If deposit_id is parsed, FE can call GET /api/deposits/{depositId}.
