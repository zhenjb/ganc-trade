P1 Checklist — MsgDeposit for Wallet Flow

This is only the checklist P1 needs to confirm before P5 implements wallet-signed deposit.

1. Confirm correct message

  • Use: proto/ob/zkdex/v1/tx.proto
  • Do not use: proto/ob/backend/v1/tx.proto
  • Package: ob.zkdex.v1
  • Type URL: /ob.zkdex.v1.MsgDeposit
  • Signer field: creator

2. Confirm MsgDeposit fields

{
  "creator": "<wallet address>",
  "denom": "uusdc",
  "amount": "100"
}

3. Confirm EventDeposit fields

  • Event type: ob.zkdex.v1.EventDeposit
  • deposit_id
  • creator
  • denom
  • amount

4. Provide chain config to P5

  • chainId
  • rpc, for example http://localhost:26657
  • rest, for example http://localhost:1317
  • bech32Prefix
  • feeDenom
  • defaultGas and defaultFee
  • Supported denoms: uatom, uusdc

5. Provide one real deposit sample

  • txHash
  • deposit_id from EventDeposit
  • creator
  • denom
  • amount
  • Result of: GET /ob/zkdex/v1/deposit_record/{deposit_id}

6. Ensure demo account is funded

  • Has fee denom, for example uatom
  • Has deposit denom, for example uusdc
